# Parti B2B ERP 功能手册

本文档根据当前代码库反向整理，适用于产品验收、演示培训和后续需求梳理。

## 1. 系统概览

Parti B2B ERP 是一个面向经销商下单、管理员运营、加工车间履约的轻量 ERP。系统当前覆盖：

- 经销商报价、保存草稿、提交订单、跟踪订单状态。
- 管理员销售订单审核、加工派单、经销商/产品/供应商/车间主数据维护。
- 采购单创建、采购收货、车间库存入库。
- 加工单状态推进、进入打包阶段时自动扣减库存。
- 车间库存查看、库存流水、盘点与盘点调整。
- 经销商应收对账、供应商应付对账、收付款登记。
- 系统设置：表面处理、加工工艺、等级折扣、成本参数、物流承运商。

## 2. 角色与入口

### 2.1 登录

入口：`/login`

用户通过邮箱和密码登录。登录成功后系统按角色跳转：

- 经销商：进入 `/dealer`
- 管理员：进入 `/admin`
- 车间用户：进入 `/workshop`

当前登录页内置快捷填入管理员和经销商演示账号；车间账号需由管理员在“加工车间”模块中创建。

### 2.2 管理员

管理员拥有运营后台权限，可访问：

- 驾驶舱
- 销售订单
- 加工制单
- 经销商
- 加工车间
- 产品目录
- 供应商
- 采购单
- 经销商对账
- 供应商对账
- 报价成本
- 系统设置

### 2.3 经销商

经销商拥有下单与订单跟踪权限，可访问：

- 工作台
- 报价下单
- 我的订单

经销商只能查看和操作自己所属经销商的数据。

### 2.4 车间用户

车间用户拥有加工和库存权限，可访问：

- 我的加工单
- 历史加工
- 库存
- 盘点

车间用户只能查看和操作自己绑定车间的数据。

## 3. 管理员功能

### 3.1 运营驾驶舱

入口：`/admin`

驾驶舱用于查看经营概览：

- 本月销售：已确认及后续状态订单金额。
- 本月收款：经销商回款金额。
- 本月采购：非取消采购单金额。
- 本月付款：供应商付款金额。
- 总应收：经销商应收余额汇总。
- 总应付：供应商应付余额汇总。
- 净现金口径：总应收减总应付。
- 在产订单数量。
- 低库存 SKU 数量。
- 订单状态统计：待审核、已确认、生产中、已完成。
- 应收 Top 5、应付 Top 5。
- 最近待审核订单列表。

### 3.2 销售订单

入口：`/admin/orders`

管理员可按状态查看销售订单。列表展示：

- 订单号
- 经销商
- 下单日期
- 交期
- 行数
- 金额
- 状态

订单详情入口：`/admin/orders/[orderNo]`

详情页包含：

- 订单基础信息。
- 经销商与收货信息。
- 订单明细和利润核算。
- 审核状态。
- 审核操作。
- 加工派单操作。

审核规则：

- 只有 `PENDING` 状态订单可审核。
- 管理员可通过或驳回。
- 审核可填写审核备注、建议交期、确认金额。
- 通过后订单进入可派单状态。

派单规则：

- 只有 `CONFIRMED` 或 `PARTIALLY_PAID` 订单可派单。
- 全部为 `OUTSOURCED` 外购行的订单无需派至车间。
- 一个销售订单只能生成一个加工单。
- 派单时需选择目标车间，可填写承诺交付日期和备注。
- 派单后销售订单进入生产相关状态。

### 3.3 加工制单

入口：`/admin/work-orders`

管理员可查看所有加工单：

- 加工单号
- 销售订单号
- 经销商
- 车间
- 状态
- 承诺交期
- 物流信息

详情入口：`/admin/work-orders/[workOrderNo]`

加工单状态流：

1. `SCHEDULED` 已排产
2. `PREPARING` 备料中
3. `PROCESSING` 加工中
4. `QC` 质检中
5. `PACKING` 打包中
6. `READY_TO_SHIP` 待出仓
7. `SHIPPED` 已出运

规则：

- 如果加工单不需要质检，可跳过 `QC`。
- 出运到 `SHIPPED` 时必须填写物流公司和运单号。
- 加工单状态会同步影响销售订单状态：
  - `SCHEDULED` 到 `PACKING`：销售订单为生产中。
  - `READY_TO_SHIP`：销售订单为待发货。
  - `SHIPPED`：销售订单为已发货。

### 3.4 经销商管理

入口：`/admin/dealers`

管理员可新增和编辑经销商：

- 经销商编号
- 公司名称
- 联系人
- 联系电话
- 价格等级：A/B/C/D/E
- 结算方式：预付、定金、信用
- 信用额度
- 状态：启用/停用

规则：

- 经销商编号唯一。
- 编辑已有经销商时，经销商编号不可修改。
- 经销商价格等级影响报价和订单价格。
- 信用结算经销商下单时会校验可用信用额度。
- 信用订单审核通过后会占用信用额度：已用额度增加、可用额度减少。
- 登记经销商收款后会释放已占用信用额度。
- 删除经销商收款记录会回滚信用释放。

### 3.5 加工车间管理

入口：`/admin/workshops`

管理员可新增和编辑车间：

- 车间编号
- 车间名称
- 负责人
- 联系电话
- 地址
- 状态：启用/停用

还可为车间创建登录用户：

- 邮箱
- 姓名
- 密码

规则：

- 车间编号唯一。
- 车间用户创建后绑定该车间，只能查看和操作本车间加工单、库存和盘点。

### 3.6 产品目录

入口：`/admin/products`

产品分为两类：

- `PROFILE` 型材
- `HARDWARE` 零配件

管理员可维护：

- SKU
- 产品名称
- 系列
- 规格
- 零售价
- 采购成本
- 单位
- 是否原料
- 原料棒长
- 生产良率
- 是否需要图纸
- 是否启用

规则：

- SKU 唯一。
- `PROFILE` 中可标记原料型材，用于经销商自定义切长订单的原料扣减。
- 原料型材可维护棒长和良率，工单进入 `PACKING` 时会按该原料 SKU 的配置计算消耗棒数。
- `HARDWARE` 的订单价格以产品目录零售价和经销商等级折扣为准，客户端传入价格仅作参考。
- 需要图纸的产品，下单时必须上传图纸。

### 3.7 供应商管理

入口：`/admin/suppliers`

管理员可新增和停用供应商：

- 供应商编号
- 名称
- 联系人
- 电话
- 地址
- 备注
- 启用状态

规则：

- 供应商编号唯一。
- 停用供应商不可用于新建采购单。

### 3.8 采购单

入口：`/admin/purchase-orders`

采购单列表展示：

- 单号
- 供应商
- 目标车间
- 下单日期
- 期望到货日期
- 行数
- 金额
- 状态

新建入口：`/admin/purchase-orders/new`

新建采购单需要：

- 供应商
- 目标车间
- 期望到货日期
- 备注
- 采购明细

采购明细可从产品目录中选择 SKU，填写采购数量和单价。

采购单状态：

- `DRAFT` 草稿
- `SENT` 已下单
- `PARTIALLY_RECEIVED` 部分收货
- `RECEIVED` 已收货
- `CLOSED` 已关闭
- `CANCELLED` 已取消

收货规则：

- 只有管理员可收货。
- 已取消或已关闭采购单不可收货。
- 收货数量不能超过未收数量。
- 每次收货会写入库存流水 `PO_RECEIPT`。
- 收货后增加目标车间库存。
- 全部收完后采购单状态变为 `RECEIVED`；部分收货则为 `PARTIALLY_RECEIVED`。

### 3.9 盘点审核

入口：`/admin/stock-counts`

管理员可查看所有车间提交的盘点单：

- 盘点单号
- 车间
- 状态
- SKU 数
- 盘盈项数
- 盘亏项数
- 提交人和提交时间
- 审核人和审核时间

审核规则：

- 只有 `SUBMITTED` 已提交盘点单可审核。
- 审核通过后，系统才会对差异写入 `STOCK_COUNT_ADJUST` 库存流水。
- 已审核盘点单不可重复写入库存调整。

### 3.10 经销商对账

入口：`/admin/reconcile/dealers`

列表展示：

- 经销商编号
- 公司名称
- 订单数
- 应收
- 已收
- 余额

详情页展示订单和收款明细，并支持登记收款。

应收规则：

- 计入应收的销售订单状态：
  - `CONFIRMED`
  - `PARTIALLY_PAID`
  - `PRODUCING`
  - `READY`
  - `SHIPPED`
  - `COMPLETED`
- 待提交、待审核、修改中、取消、驳回订单不计入应收。
- 余额 = 应收订单金额合计 - 已登记收款。

收款核销规则：

- 管理员登记经销商收款后，系统会按订单日期从早到新自动核销未收订单。
- 核销会写入收款分摊明细，并更新销售订单的 `paidAmount` 和 `paymentStatus`。
- 部分核销时订单付款状态为 `PARTIAL`，全额核销时为 `PAID`。
- 信用经销商收款会同步释放已占用信用额度。
- 删除收款记录会删除对应核销明细，并回滚订单已收金额、付款状态和信用释放。

### 3.11 供应商对账

入口：`/admin/reconcile/suppliers`

列表展示：

- 供应商编号
- 名称
- PO 数
- 应付
- 已付
- 余额

详情页展示采购单和付款明细，并支持登记付款。

应付规则：

- 已取消采购单不计入应付。
- 应付金额按“已收货数量 × 单价”计算，不按采购单下单总额计算。
- 余额 = 已收货应付金额 - 已登记付款。

付款核销规则：

- 管理员登记供应商付款后，系统会按采购单日期从早到新自动核销已收货未付采购单。
- 核销会写入付款分摊明细，并更新采购单 `paidAmount`。
- 删除供应商付款记录会删除对应核销明细，并回滚采购单已付金额。

### 3.12 报价成本

入口：`/admin/pricing`

管理员可输入长度和经销商等级，查看报价成本分析：

- 理论重量
- 损耗重量
- 实际重量
- 材料成本
- 加工成本
- 表面处理成本
- 连接件成本
- 总成本
- 零售价
- 含税价
- 各等级定价

默认定价参数包括：

- 型材米重
- 截断利用率
- 素材价格
- 加工费
- 表面处理费
- 连接件费
- 毛利率
- 一级/二级代理折扣
- 含税加成

### 3.12 系统设置

入口：`/admin/settings`

管理员可维护：

- 表面处理工艺选项。
- 表面颜色选项。
- 加工操作选项。
- 加工规格/修饰选项。
- 经销商等级折扣。
- 成本与定价参数。
- 物流承运商。

这些配置会影响经销商报价下单、报价计算和发货物流选择。

## 4. 经销商功能

### 4.1 经销商工作台

入口：`/dealer`

工作台展示：

- 经销商公司名称。
- 经销商编号。
- 价格等级。
- 结算方式。
- 信用额度。
- 可用额度。
- 待审核订单数。
- 生产中订单数。
- 最近订单。

快捷入口：

- 报价下单
- 我的订单

### 4.2 报价下单

入口：`/dealer/quote`

经销商可创建多行订单，支持三种行类型：

- `PROFILE` 型材
- `HARDWARE` 零配件
- `OUTSOURCED` 外购

#### 型材行

可填写或选择：

- 原料型材
- 长度
- 表面工艺
- 颜色
- 加工操作
- 图纸
- 数量
- 目标价比例

规则：

- 型材行必须引用一个有效的原料型材。
- 可上传图纸。
- 支持自定义切长。

#### 零配件行

可填写或选择：

- SKU
- 名称/规格
- 数量
- 目标价比例
- 图纸

规则：

- 零配件必须关联产品目录中的 `HARDWARE` 产品。
- 如果产品要求图纸，则必须上传图纸。
- 服务端会按产品零售价和经销商等级折扣重新计算单价。

#### 外购行

可填写：

- 名称
- 规格
- 图纸
- 数量
- 采购单价
- 目标售价

规则：

- 外购行参与订单金额。
- 外购行不会生成车间库存消耗。
- 全部为外购的订单无需派给车间。

#### 收货信息

经销商可选择已有地址，也可新填地址。

#### 提交方式

- 保存草稿：订单为 `DRAFT`。
- 提交审核：订单进入 `PENDING`。

信用额度规则：

- 如果经销商结算方式为 `CREDIT`，系统会校验可用额度是否足够覆盖订单金额。
- 信用额度不足时不能提交。
- 管理员审核通过信用订单时会再次校验可用额度，防止草稿到审核期间额度被其他订单占用。
- 若经销商档案开启“允许超信用额度下单”，则可跳过额度不足拦截，但仍会记录已用额度。

订单状态包括：

- `MODIFYING` 待确认
- `CONFIRMED` 已确认
- `PARTIALLY_PAID` 部分付款
- `PRODUCING` 生产中
- `READY` 待发货
- `SHIPPED` 已发货
- `COMPLETED` 已完成

## 5. 车间功能

### 5.1 我的加工单

入口：`/workshop`

车间用户可查看本车间未出运的加工单：

- 加工单号
- 销售单号
- 收货人
- 承诺交期
- 品项数量
- 状态

列表按承诺交期排序，逾期加工单会突出显示。

### 5.2 加工单详情与推进

入口：`/workshop/orders/[workOrderNo]`

车间用户可查看加工单详情，并推进状态。

规则：

- 车间用户只能操作本车间加工单。
- 加工单状态按固定流程推进。
- 推进到 `PACKING` 时，系统自动扣减库存：
  - `HARDWARE` 行按 SKU 和数量 1:1 扣减。
  - `PROFILE` 行按原料型材扣减棒数。
  - `OUTSOURCED` 行不扣减库存。
- 同一个加工单进入 `PACKING` 的库存扣减具备幂等保护，不会重复扣减。

型材原料棒扣减规则：

```text
棒数 = ceil(切长合计 mm / 原料棒长 mm / 原料良率)
```

其中：

- 原料棒长来自产品目录中该原料型材的 `lengthMm`。
- 原料良率来自产品目录中该原料型材的 `yieldRate`。
- 未配置时使用默认值：棒长 3600mm，良率 95%。

### 5.3 历史加工

入口：`/workshop/history`

展示本车间已出运加工单：

- 加工单号
- 销售单号
- 收货人
- 出运时间
- 物流
- 状态

### 5.4 车间库存

入口：`/workshop/inventory`

库存页包含两部分：

- 当前现货。
- 最近 50 条库存流水。

当前现货展示：

- SKU
- 名称
- 现货数量
- 低库存阈值
- 状态
- 最近更新时间

库存流水展示：

- 时间
- SKU
- 名称
- 流水类型
- 变动数量
- 结存
- 关联单据
- 备注

库存流水类型：

- `PO_RECEIPT` 采购入库
- `WORK_ORDER_CONSUME` 工单消耗
- `STOCK_COUNT_ADJUST` 盘点调整
- `MANUAL_ADJUST` 手工调整

库存控制规则：

- 库存变动默认禁止产生负库存。
- 工单进入 `PACKING` 时，如任一零配件或原料型材库存不足，状态推进会失败并提示不足 SKU。
- 盘点审核时，如调整会导致当前库存为负，审核会失败并提示不足 SKU。
- 未来如需特殊负库存场景，必须由代码显式开启 `allowNegative`，默认业务操作不会放行。

### 5.5 库存盘点

入口：`/workshop/stock-count`

车间用户可发起盘点。系统会对当前车间库存做快照，生成盘点单。

盘点单详情入口：`/workshop/stock-count/[countNo]`

盘点流程：

1. 发起盘点。
2. 系统生成草稿盘点单，明细默认实盘数等于系统数。
3. 车间修改每个 SKU 的实盘数。
4. 系统自动计算差异。
5. 提交盘点，盘点单状态变为 `SUBMITTED`。
6. 管理员在“盘点审核”中审核。
7. 审核通过后，系统对所有差异写入 `STOCK_COUNT_ADJUST` 库存流水，盘点单状态变为 `APPROVED`。

规则：

- 只有 `DRAFT` 草稿盘点单可编辑和提交。
- `SUBMITTED` 已提交盘点单不可再由车间编辑，需等待管理员审核。
- 当前无库存项时不能发起盘点。
- 车间用户只能操作本车间盘点单。

## 6. 业务主流程

### 6.1 标准销售履约流程

1. 管理员维护产品、经销商、车间。
2. 经销商进入“报价下单”创建订单。
3. 经销商保存草稿或提交审核。
4. 管理员在“销售订单”审核订单。
5. 审核通过后管理员派单到车间。
6. 车间在“我的加工单”查看加工任务。
7. 车间推进加工单状态。
8. 进入 `PACKING` 时系统扣减库存。
9. 出运后填写物流信息，销售订单同步为已发货。
10. 管理员登记经销商收款，对账余额更新。

### 6.2 标准采购入库流程

1. 管理员维护供应商和产品目录。
2. 管理员创建采购单，指定供应商和目标车间。
3. 管理员在采购单详情中登记收货数量。
4. 系统写入采购入库流水。
5. 目标车间库存增加。
6. 供应商对账按已收货金额形成应付。
7. 管理员登记供应商付款，对账余额更新。

### 6.3 标准库存盘点流程

1. 车间进入“盘点”。
2. 点击发起盘点。
3. 系统快照当前库存。
4. 车间录入实盘数。
5. 提交盘点。
6. 管理员审核盘点单。
7. 系统写入盘点调整流水。
8. 库存数量按差异调整。

## 7. 权限规则

- 未登录用户会被重定向到登录页。
- 经销商不能访问管理员后台。
- 管理员不能使用经销商下单接口。
- 车间用户只能访问和操作绑定车间的数据。
- 经销商只能访问自己的订单和地址。
- 产品、供应商、车间、经销商、采购、收付款维护均要求管理员权限。
- 盘点发起要求车间用户权限。
